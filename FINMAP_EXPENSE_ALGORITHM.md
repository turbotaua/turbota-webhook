# Алгоритм "Звірка витрат Finmap"

## Розклад
Кожні 2 дні, автоматично. Результат — звіт в групу "Облік".

## КРОК 0 — Отримати непозначені витрати

Використати `finmap_log_controller_get_list` (POST /v2.2/operations/list):
- `dateFrom` / `dateTo` = останні 2 дні
- `type` = expense
- Фільтр: `categoryId = null` або порожній

З кожної транзакції витягнути:
- **Суму** (точну, напр. 9355.50)
- **Коментар** (там зазвичай контрагент: "ФОП Наджафов Ельвін; рахунок 35124")
- **Рахунок** (ФОП mono Молодій / ФОП mono Панькевич)
- **Дату**

| Паттерн | Категорія | Авто? |
|---------|-----------|-------|
| "На білу картку" + Молодій + ~10 000 грн | Зарплата → Настя (співзасновниця) | ДА |
| "На білу картку" / "На чорну картку" (інші суми) | Transfer між рахунками (НЕ витрата!) | ДА |
| ~10 000 грн + Панькевич + "На білу/чорну картку" | Зарплата → Назарій Панькевич | ДА |
| "BASE44", "Claude", "Google", "Notion", "OpenAI" | Витрати на підписки | ДА |
| "Нова пошта" | Доставка товару клієнтам | ДА |
| "Комісія банку" | Комісія банку | ДА |
| "Наджафов Ельвін" або "Msperfum" | Матеріали — але ЩО саме? | **НІ — йди до кроку 2** |
| "Тенюх-Мармура" | Оплата постачальникам готової продукції | ДА |

**Якщо повний збіг → готово, далі не йти.**

## КРОК 2 — Пошук по Telegram чату в Supabase (0 кредитів)

**Це ГОЛОВНЕ джерело.** Система компанії: кожну витрату працівник кидає в чат "облік".

### SQL запити до `telegram_messages_log`:

#### 2.1 — Пошук по ТОЧНІЙ СУМІ:
```sql
SELECT text, from_name, from_username, topic_id, date
FROM telegram_messages_log
WHERE (
    9355.50 = ANY(amounts)           -- точна сума
    OR text ILIKE '%9355%'           -- сума в тексті
    OR text ILIKE '%9 355%'          -- з пробілом
)
AND date > now() - interval '5 days'
ORDER BY date DESC;
```

#### 2.2 — Пошук по імені контрагента з коментаря транзакції:
```sql
-- Якщо коментар = "ФОП Наджафов Ельвін"
SELECT text, from_name, topic_id, date
FROM telegram_messages_log
WHERE (
    text ILIKE '%Наджафов%'
    OR text ILIKE '%Msperfum%'
    OR text ILIKE '%msp%'
)
AND date > now() - interval '5 days'
ORDER BY date DESC;
```

#### 2.3 — Пошук по ключових словах:
```sql
-- Якщо шукаємо мед
SELECT text, from_name, topic_id, date
FROM telegram_messages_log
WHERE 'мед' = ANY(keywords)
AND date > now() - interval '5 days';
```

### Зіставлення знайденого:

**ПРОЧИТАТИ повідомлення і зрозуміти КОНТЕКСТ:**
- Хто написав?
- В якому топіку? ("закупівля і прихід" = закупка сировини/матеріалів)
- Що саме замовляли/купували?
- Чи збігається сума?

**Приклади зіставлення:**
- Знайшов "Закупівля меду 3700 грн" → категорія = Закупівля сировини / Мед
- Знайшов "Msperfum банки Linda 9355.50" → категорія = Матеріали / Банки Linda
- Знайшов "оплата за свічки Тенюх-Мармура 5760" → Оплата постачальникам

## КРОК 3 — Що бот сам вносив (0 кредитів)

Перевірити свою пам'ять / conversation history:
- "Я сьогодні вносив закупівлю від Msperfum на 9355.50 в Dilovod"
- "Я вносив прихід меду від Панькевич на 3700"

**Правило:** Якщо бот сам в ці ж дні вносив документ в Dilovod з тією ж сумою і контрагентом — це 100% та сама операція. Категорія визначається по типу документа.

---

## КРОК 4 — Finmap історія по контрагенту (1 запит)

Якщо в коментарі є ім'я ФОП/ТОВ:
- `finmap_log_controller_get_list` з фільтром по тому ж коментарю за останні 90 днів
- Подивитися яка категорія стояла на попередніх оплатах тому ж контрагенту
- Якщо 5 з 5 = "Аромати" → з високою ймовірністю це знову "Аромати"

⚠️ АЛЕ: Один контрагент може продавати різне! Finmap history дає лише **підказку**, а не фінальну відповідь.

## КРОК 6 — Формування звіту

### Структура:
```
💸 Звірка витрат | [дати]

✅ Визначено автоматично:
• [сума] — [рахунок] — [коментар] → [категорія] (джерело: [авто-правило/чат/finmap])

🔄 Transfer (не витрата):
• [сума] — [рахунок] — переказ між рахунками

❓ Потребують уточнення:
• [сума] — [рахунок] — [коментар] — [що знайшов але не впевнений]

Підтвердіть ✅ та уточніть ❓ → позначу в Finmap.
```

### Правила звіту:
- Коротко, по-людськи, максимально стисло
- Вказувати ДЖЕРЕЛО визначення (авто-правило / знайшов в чаті / finmap history)
- Transfer — виділяти ОКРЕМО, це НЕ витрата
- Якщо Молодій → @lmolodiy + можна пожартувати
- Якщо Панькевич → @harunder
- Перша звірка дня — можна написати щось мотиваційне

## КАРТА КОНТРАГЕНТІВ

Зберігається в Supabase: `finmap_counterparties_cache`

```sql
SELECT finmap_comment_pattern, real_name, sells, default_category, notes
FROM finmap_counterparties_cache;
```

Бот може сам додавати нових контрагентів через INSERT коли вивчає нові паттерни.

## АНТИ-ПАТТЕРНИ (чого НЕ робити)

- ❌ НЕ питати "що це?" якщо ти сам це вносив сьогодні
- ❌ НЕ питати якщо сума є в чаті з поясненням — прочитай і зрозумій
- ❌ НЕ ставити категорію "Аромати" на Наджафова без уточнення — він продає різне
- ❌ НЕ ходити в Dilovod першим ділом — це дорого, спочатку чат
- ❌ НЕ ігнорувати топіки — "закупівля і прихід" це підказка що це закупка
- ❌ НЕ відправляти звіт без пошуку по чату — це виглядає тупо
- ❌ НЕ плутати Transfer з витратою — "на чорну картку" це переказ
